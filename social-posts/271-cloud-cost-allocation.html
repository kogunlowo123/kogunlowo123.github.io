The Real Trade-offs

  • Auto-deletion is the nuclear option: It works because it creates real consequences. But deleting production resources because someone forgot a tag is a career-ending incident. Start with stop, not terminate. Add a 48-hour warning period.
  • Prevention beats remediation: Use SCPs or IAM policies to prevent resource creation without required tags. Block at the API level, not after the fact. This is cheaper and safer than cleanup automation.
  • Not all resources support tags: Some AWS resources still don't support tags well. Data transfer, support charges, and certain managed services are difficult to attribute. Accept that 100% tagging is aspirational.
  • IaC should enforce tags automatically: Terraform modules, CloudFormation defaults, and CDK constructs should include required tags by default. If tags come from code, humans don't forget them.
  • Tag hygiene degrades over time: Even with enforcement, tag values drift — teams rename, projects end, owners leave. Regular tag audits (monthly) are necessary to maintain allocation accuracy.
