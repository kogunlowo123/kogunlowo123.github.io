Debezium Is the Most Underrated Data Tool

Change Data Capture with Debezium turns your database transaction log into a real-time event stream. No polling, no dual writes, no application changes. It captures every INSERT, UPDATE, and DELETE and streams it to Kafka. AWS DMS does something similar but with less flexibility and more cost.

The Real Trade-offs

  • Debezium requires Kafka infrastructure: You need Kafka and Kafka Connect running to use Debezium. DMS is fully managed — no infrastructure to operate. For teams without Kafka expertise, DMS wins on operational simplicity.
  • Debezium captures schema changes: It streams DDL changes alongside data changes, enabling full schema evolution tracking. DMS focuses on data replication and handles schema changes less gracefully.
  • DMS is a migration tool first: DMS was built for database migrations with ongoing replication as a secondary feature. Debezium was purpose-built for event streaming from databases.
  • Exactly-once semantics are hard: Both tools deliver at-least-once by default. Achieving exactly-once requires careful consumer-side deduplication regardless of which tool you choose.
  • Debezium's connector ecosystem is broader: PostgreSQL, MySQL, MongoDB, SQL Server, Oracle, Cassandra, and more. DMS supports many databases too, but Debezium's connectors are more configurable.
