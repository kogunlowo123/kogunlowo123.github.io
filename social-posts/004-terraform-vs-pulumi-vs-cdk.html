Terraform's HCL Is a Dead-End Language

HCL is not a real programming language. No loops (for_each is a hack), no proper conditionals, no unit testing framework. Pulumi and CDK let you use TypeScript, Python, Go — real languages with real tooling. Why are we still writing HCL in 2026?

The Real Trade-offs

  • Terraform has the largest ecosystem and community — but HCL forces workarounds for basic logic
  • Pulumi gives you real programming but state management is less mature than Terraform
  • AWS CDK is powerful but locks you into CloudFormation limits (500 resource cap, slow rollbacks)
  • Terraform's state file is a single point of failure that every team struggles with
  • HCL has no package manager, no IDE refactoring support, and no unit testing — Pulumi has all three
