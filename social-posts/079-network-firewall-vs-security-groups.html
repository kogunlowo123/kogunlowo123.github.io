Security Groups Are Not a Firewall

Security Groups are stateful L3/L4 filters — they allow or deny traffic by IP and port. They can't inspect payloads, detect intrusions, block malware, or filter DNS. Calling them a "firewall" is like calling a screen door a security system.

The Real Trade-offs

  • Security Groups are sufficient for many workloads: If you just need "only port 443 from the load balancer," SGs do the job perfectly. Not every workload needs deep packet inspection.
  • Network Firewall costs are significant: AWS Network Firewall starts at ~$900/month per endpoint. Azure Firewall Standard is ~$912/month. That's before data processing charges.
  • Egress filtering is the gap: Security Groups allow all outbound by default and can't filter by domain name. A compromised instance can exfiltrate data to any IP. Network Firewalls can filter by FQDN.
  • IDS/IPS detects what SGs miss: A network firewall with Suricata rules (AWS) or IDPS (Azure Premium) can detect C2 traffic, exploit attempts, and lateral movement that L4 filtering can't see.
  • Hub-and-spoke architecture: Centralized firewall in a hub VNet/VPC with spoke traffic inspection provides consistent policy across your entire cloud estate. SGs are per-instance and unmanageable at scale.
