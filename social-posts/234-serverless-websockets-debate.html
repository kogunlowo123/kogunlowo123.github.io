Serverless WebSockets: API Gateway vs AppSync vs IoT Core

WebSockets on serverless sounds contradictory — persistent connections on ephemeral compute. Yet AWS offers three managed options. API Gateway WebSocket API gives you raw control. AppSync handles GraphQL subscriptions automatically. IoT Core scales MQTT to millions. Each has quirks you'll only discover in production.

The Real Trade-offs

  • API Gateway WebSocket requires connection management: You must store connection IDs in DynamoDB, manage $connect/$disconnect routes, and call the Management API to push messages. It's powerful but you're building the pub/sub layer yourself.
  • AppSync subscriptions are automatic but opinionated: Define a GraphQL subscription, and AppSync handles connection management, filtering, and fan-out. But you're locked into GraphQL, and complex authorization rules on subscriptions can be tricky.
  • IoT Core scales to millions of connections: Designed for millions of IoT devices, but works perfectly for web/mobile apps. MQTT topics provide natural pub/sub, and the rules engine routes messages to Lambda, DynamoDB, or S3 without custom code.
  • Connection limits and idle timeouts: API Gateway WebSocket has a 2-hour idle timeout and 10-minute message timeout. AppSync subscriptions disconnect after 24 hours. IoT Core keeps connections alive indefinitely with MQTT keep-alive pings.
  • Cost at 100K concurrent connections: API Gateway WebSocket: ~$3.60/hr in connection minutes + message costs. IoT Core: ~$1.25/hr in connection minutes. AppSync: $0.08/M connection minutes. For high-concurrency scenarios, AppSync and IoT Core are significantly cheaper.
