Memcached Should Have Been Deprecated Years Ago

Redis supports every data structure imaginable, has persistence, pub/sub, Lua scripting, clustering, and streams. Memcached is a key-value store from 2003 that stores strings. Why are we still pretending there's a debate here?

The Real Trade-offs

  • Memcached is simpler by design: Multi-threaded, no persistence overhead, pure cache. For simple string caching at massive scale, Memcached can be more memory-efficient because there's no data structure overhead.
  • Multi-threading matters: Memcached uses multiple cores natively. Redis was single-threaded for I/O until version 6.0, and even now only uses threads for network I/O, not command execution.
  • Redis feature creep is real: Redis became a database, message broker, search engine, and time-series store. When everything is Redis, your Redis becomes a single point of failure for everything.
  • Redis licensing changed: Redis moved to a dual license (RSALv2/SSPLv1), spawning Valkey. Memcached's BSD license has no such risk.
  • For pure caching, the difference is marginal: If you only need "set key, get key, TTL" — Memcached is lighter. But the moment you need sorted sets, geospatial indexes, or streams, Redis is the only option.
