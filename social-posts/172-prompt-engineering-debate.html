Prompt Engineering Isn't Engineering — It's Guessing

Real engineering is deterministic, testable, and reproducible. Prompt "engineering" is trying different phrasings until the model gives the output you want. It breaks between model versions, doesn't transfer across providers, and has no formal verification. We're calling copywriting "engineering" because it sounds more impressive.

The Real Trade-offs

  • Structured prompting is a skill: Chain-of-thought, few-shot examples, and system prompts do measurably improve outputs. Calling it "guessing" dismisses legitimate techniques with proven results.
  • Reproducibility is improving: Temperature 0, structured outputs (JSON mode), and seed parameters make prompts more deterministic. It's not fully reproducible, but it's getting closer.
  • Model updates break prompts: A prompt optimized for GPT-4-0613 may not work with GPT-4-turbo. This fragility doesn't exist in traditional software engineering and is a legitimate concern.
  • Evals are the engineering part: Building automated evaluation suites that test prompt outputs against ground truth — that's engineering. Writing prompts without evals is indeed guessing.
  • The title will evolve: "Prompt engineering" will likely merge into general software engineering as models improve at understanding intent. The skill becomes less about phrasing and more about system design.
