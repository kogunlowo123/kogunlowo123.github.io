SD-WAN Vendors Are Obsolete — Cloud Networking Won

Cisco, VMware, and Fortinet built SD-WAN for a world where branches connected to data centers. Now branches connect to SaaS and cloud. AWS Cloud WAN, Azure Virtual WAN, and SASE platforms are eating SD-WAN's lunch. The traditional vendors just haven't admitted it yet.

The Real Trade-offs

  • Cloud-native WAN integrates natively: AWS Cloud WAN connects VPCs, VPNs, and Direct Connect in one policy. No appliances to deploy, no firmware to patch, no license renewals. The cloud provider IS the network.
  • SD-WAN still owns the branch: Cloud WAN solutions don't ship branch hardware. You still need CPE devices for office connectivity. SD-WAN vendors excel at branch-to-cloud and branch-to-branch with QoS, application-aware routing, and WAN optimization.
  • SASE is the real disruptor: Zscaler, Cloudflare, and Palo Alto's Prisma SASE combine SD-WAN with cloud security. This converged model makes both traditional SD-WAN and cloud-only networking feel incomplete.
  • Multi-cloud is SD-WAN's last stand: If you're on AWS, Azure, AND GCP, no cloud-native WAN connects all three. SD-WAN vendors and platforms like Aviatrix fill this gap — for now.
  • The migration pain is real: Enterprises with deployed SD-WAN fabrics have sunk costs in hardware, training, and operations. Ripping it out for cloud-native networking requires a compelling ROI — and the cloud solutions aren't always cheaper.
