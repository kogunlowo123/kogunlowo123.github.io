Configuration Management Is Dead — Long Live Containers

Ansible, Chef, Puppet — these tools solved the "configure 1000 servers consistently" problem. Containers solved it differently: don't configure servers at all. Package everything into an image and ship it. If 80% of workloads are containerized, who still needs config management?

The Real Trade-offs

  • Containers didn't kill config management — they shrunk its scope: Ansible is still the best tool for network device configuration, bare metal provisioning, and managing legacy VMs that can't be containerized.
  • Node configuration still exists: Even in a Kubernetes world, someone configures the nodes — kubelet settings, kernel parameters, container runtime config. That's still config management territory.
  • Dockerfiles are the new playbooks: A Dockerfile is configuration management for a single container. The shift is from managing server fleets to managing image pipelines.
  • Ansible is evolving: Ansible now has Kubernetes, cloud, and network modules. It's less about server config and more about orchestration and automation of everything.
  • The hybrid reality: Most enterprises have 60% containers, 30% VMs, and 10% bare metal. Config management isn't dead — it's just not the only tool anymore.
