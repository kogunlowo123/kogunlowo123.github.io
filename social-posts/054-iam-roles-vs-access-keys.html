If You're Using Access Keys in 2024, You're Doing It Wrong

Long-lived access keys are the #1 cause of AWS breaches. They get committed to Git, shared in Slack, and never rotated. IAM roles with temporary credentials exist. Use them.

The Real Trade-offs

  • Access keys are simple: Copy-paste AKID and secret, and your script works. IAM roles require understanding STS, trust policies, and session tokens. Simplicity has value.
  • CI/CD still needs credentials sometimes: OIDC federation for GitHub Actions is great, but self-hosted Jenkins, legacy tools, and third-party integrations often require static keys.
  • Rotation != elimination: Even with 90-day rotation policies, keys exist for 90 days. That's 90 days of exposure. STS tokens expire in 1 hour by default.
  • Developer experience matters: Local development with IAM roles requires AWS SSO, credential helpers, and profile management. Access keys are frictionless. Friction reduces adoption.
  • The real fix is identity federation: Whether it's IAM Identity Center, Entra workload identity, or GCP Workload Identity — federate everything and keys become unnecessary.
