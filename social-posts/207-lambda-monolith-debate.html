Lambda Monoliths vs Micro-Functions

Lambda monoliths outperform micro-functions. A single Lambda with internal routing (Express, Fastify) has better cold start performance, shared connection pools, and simpler deployments than 50 individual functions.

The Real Trade-offs

  • Cold starts are amortized in monoliths: One warm container serves all routes. With micro-functions, infrequently called endpoints always cold start. A monolith's hot path keeps the container warm for everything.
  • Connection pool sharing is massive: A monolith Lambda maintains one database connection pool. Fifty micro-functions each create their own, potentially exhausting database connections.
  • Micro-functions win on IAM granularity: Each function can have minimal permissions. A monolith Lambda needs permissions for everything — violating least-privilege principles.
  • Deployment complexity scales differently: Deploying one function is simple. Coordinating deployments of 50 interdependent functions with shared API Gateway stages is a CI/CD nightmare.
  • The sweet spot is domain-bounded Lambdas: Not one mega-function and not one per route. Group by domain — users-service, orders-service, payments-service — each with internal routing.
