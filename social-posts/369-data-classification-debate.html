Data Classification is the Hardest Easy Problem in Security

Every security framework starts with "classify your data." Sounds simple: Public, Internal, Confidential, Restricted. But when a Slack message contains a customer's email, a meeting note references revenue numbers, and a debug log captures PII — what classification applies? Data classification at scale is unsolvable without automation.

The Real Trade-offs

  • Manual classification doesn't scale: Asking employees to classify every document, email, and file creates friction and inconsistency. People guess, skip, or mis-classify — making the entire scheme unreliable.
  • Automated discovery finds what you missed: Macie, Purview, and Cloud DLP can scan petabytes of data for PII patterns. They find sensitive data in places you never expected — log files, database backups, temporary storage.
  • Classification without enforcement is pointless: Labeling data as "Confidential" means nothing if access controls don't change. Classification must drive IAM policies, encryption requirements, and retention rules automatically.
  • Data changes classification over time: A draft document is internal; once it contains customer data, it's confidential. Aggregated anonymized data might be public. Static labels can't capture dynamic sensitivity.
  • Counter-argument — perfect classification is unnecessary: Treat all data as sensitive by default. Encrypt everything, restrict access to least privilege, and log all access. You don't need classification if your baseline security is high enough.
