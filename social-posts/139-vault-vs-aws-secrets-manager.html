Vault vs AWS Secrets Manager: The Secrets Sprawl Nobody Admits

HashiCorp Vault is the Swiss Army knife of secrets management — dynamic secrets, PKI, transit encryption, SSH certificates. AWS Secrets Manager stores and rotates secrets. One requires a dedicated team to operate. The other just works. But "just works" has a ceiling.

The Real Trade-offs

  • Vault's operational burden is real: Unsealing, HA configuration, storage backends, audit logging, policy management — operating Vault is a full-time job. HCP Vault Cloud reduces this, but at significant cost.
  • Dynamic secrets change the security model: Vault generates unique, time-limited database credentials per application instance. Secrets Manager rotates shared credentials on a schedule. Dynamic secrets mean a compromised credential expires in hours, not months.
  • AWS lock-in is the hidden cost: Secrets Manager integrates seamlessly with RDS, Lambda, ECS. Moving to another cloud means migrating every secret reference. Vault works identically across all clouds.
  • Most teams don't need Vault: If you're AWS-only with RDS databases and static API keys, Secrets Manager with rotation Lambdas covers 90% of use cases at a fraction of the operational cost.
  • The real risk is secrets sprawl: Some secrets in Vault, others in AWS Secrets Manager, some in Parameter Store, a few in environment variables. The proliferation of secret stores is a bigger risk than the choice of any single one.
