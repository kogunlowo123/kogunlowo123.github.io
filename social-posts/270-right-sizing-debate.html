The Real Trade-offs

  • Right-size BEFORE you commit: Buying RIs or Savings Plans before right-sizing locks in waste. Downsize first, observe for 2 weeks, then commit. The order matters enormously.
  • Most instances are 2-4x oversized: Studies consistently show 40-60% of cloud instances are oversized. Downsizing from m5.2xlarge to m5.large saves 75% — more than any RI discount.
  • Memory is the hidden constraint: CloudWatch doesn't collect memory metrics by default. Teams right-size based on CPU alone and crash when memory runs out. Install the CloudWatch Agent first.
  • Graviton is right-sizing's best friend: Moving from x86 to Graviton (ARM) provides 20% better price-performance. Combined with right-sizing, you can achieve 50-80% savings without any commitment.
  • Automation makes it sustainable: Manual right-sizing is a one-time exercise. Automated tools like AWS Compute Optimizer, Spot.io, or custom Lambda functions make it continuous.
