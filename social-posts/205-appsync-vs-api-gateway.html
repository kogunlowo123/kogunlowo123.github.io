AppSync vs API Gateway: AWS's Most Underrated Service

AppSync is AWS's most underrated service. Real-time subscriptions over WebSockets, offline data sync, and direct DynamoDB resolvers without Lambda — yet most teams default to API Gateway and rebuild all of this from scratch.

The Real Trade-offs

  • Direct DynamoDB resolvers eliminate Lambda: For CRUD operations, AppSync resolves directly to DynamoDB — no Lambda invocation, no cold starts, lower latency, lower cost.
  • Real-time subscriptions are built in: WebSocket-based subscriptions work out of the box. With API Gateway, you need to build and manage a WebSocket API separately.
  • VTL resolvers are the pain point: Velocity Template Language is the reason many teams abandon AppSync. The new JavaScript resolver runtime fixes this, but the legacy VTL docs still dominate search results.
  • GraphQL isn't always the right choice: If your API is simple CRUD with no relationships, REST is simpler. AppSync adds complexity that only pays off with complex, interconnected data models.
  • Offline sync is a mobile game-changer: AppSync + Amplify DataStore gives you conflict resolution and offline-first capabilities that would take months to build on API Gateway.
