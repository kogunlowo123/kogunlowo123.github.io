Your SIEM Costs More Than Your Security Team — Time for a Data Lake?

Splunk charges per GB ingested. At 500GB/day, you're looking at $1M+ annually. Amazon Security Lake stores the same data in S3 for pennies on the dollar. But a data lake doesn't alert, correlate, or investigate — it just stores. Can you build a modern SOC on cheap storage and custom queries, or do you still need a real SIEM?

The Real Trade-offs

  • Cost difference is 10-50x: Storing 1TB/day in S3 costs ~$700/month. The same data in Splunk costs $50,000+/month. For long-term retention and compliance, security data lakes are economically unbeatable.
  • SIEMs provide out-of-box detection: Sentinel ships with 300+ analytics rules, UEBA, and fusion detection. A data lake gives you raw storage — you need to build every detection rule, correlation engine, and alert pipeline from scratch.
  • Threat hunting favors data lakes: When you need to search 12 months of DNS logs for a specific IOC, a data lake query costs cents. In Splunk, that same search across archived data requires expensive license capacity or slow summary indexes.
  • Real-time alerting needs a SIEM: Data lakes are optimized for batch analytics, not sub-second alerting. When an active attacker is in your network, you need real-time correlation — a 5-minute query delay could mean the difference between containment and breach.
  • The hybrid approach is emerging: Hot data (7 days) in SIEM for real-time detection. Warm data (90 days) in data lake for investigation. Cold data (1+ year) in glacier storage for compliance. The question is whether you have the engineering team to build and maintain this architecture.
