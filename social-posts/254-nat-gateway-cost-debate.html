The Real Trade-offs

  • The math is brutal: 3 NAT Gateways (one per AZ) running 24/7 = $97/month just for hours. Process 10TB/month and add $450 in data charges. That's $547/month before any EC2 spend.
  • NAT instances are back: A t4g.nano costs $3/month. With proper iptables config and auto-recovery, it handles moderate traffic fine. Yes, it's a single point of failure per AZ, but for dev/staging, who cares?
  • VPC endpoints eliminate most NAT traffic: S3, DynamoDB, ECR, CloudWatch, STS — all can use VPC endpoints. Gateway endpoints for S3 and DynamoDB are free. Most NAT traffic is API calls to AWS services.
  • The managed service argument: NAT Gateway is fully managed, scales to 100 Gbps, and requires zero maintenance. For production workloads processing real traffic, the reliability premium is justified.
  • The real fix: audit your traffic: VPC Flow Logs can show exactly what's going through NAT. Most teams find 70%+ is S3 and ECR pulls — traffic that should use endpoints instead.
