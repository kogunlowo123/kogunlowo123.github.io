The Real Trade-offs

  • Immutability solves Requirement 2 and 6: Container images are immutable — no configuration drift, no unauthorized changes. This simplifies hardening (Req 2) and change management (Req 6) dramatically.
  • Network policies enforce microsegmentation: Kubernetes network policies can isolate the CDE at the pod level, providing finer-grained segmentation than traditional VLANs and firewalls.
  • Image scanning meets Requirement 5: Automated vulnerability scanning in CI/CD pipelines and registries provides continuous malware and vulnerability detection without traditional antivirus agents.
  • QSAs struggle with container scope: Many QSAs don't understand container boundaries. Is the host in scope? The orchestrator? The registry? Scope definition arguments can extend audit timelines significantly.
  • Counter-argument — containers add shared kernel risk: Containers share the host kernel. A kernel exploit in one container can compromise the entire node. VMs provide stronger isolation for truly sensitive cardholder data.
