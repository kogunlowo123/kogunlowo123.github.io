AWS Network Firewall vs Security Groups: Centralized vs Distributed Filtering

Security groups are stateful, free, and attached to every ENI. AWS Network Firewall runs Suricata-compatible IDS/IPS rules at the VPC level for $0.395/hr per endpoint plus $0.065/GB processed. The question isn't which is better — it's whether your compliance requirements justify a $300+/month centralized firewall.

The Real Trade-offs

  • Security groups cover 90% of use cases: For most workloads, security groups restricting inbound to specific ports from known CIDR ranges provide sufficient protection. They're free, scale automatically, and require no infrastructure management. Don't add Network Firewall just because it exists.
  • Network Firewall is for compliance-driven inspection: PCI-DSS, HIPAA, and FedRAMP often require IDS/IPS and domain-based filtering. Network Firewall with Suricata rules satisfies auditors. If compliance doesn't mandate deep packet inspection, you're paying $300+/month for peace of mind.
  • Domain-based filtering is the unique differentiator: Security groups can't filter by domain name — only IP addresses. If you need to allow traffic only to *.amazonaws.com or block known malicious domains, Network Firewall's domain list rules are the only native AWS option.
  • TLS inspection adds cost and complexity: Network Firewall can decrypt and inspect HTTPS traffic, but this requires certificate management, adds latency, and increases processing costs. Most teams enable TLS inspection only for egress traffic to catch data exfiltration.
  • Third-party alternatives may be cheaper: Palo Alto, Fortinet, and Check Point offer firewall appliances on AWS Marketplace that may provide better rule management and visibility at comparable or lower cost — especially if your team already knows those platforms.
