The Real Trade-offs

  • Convenience has value: One API call, no GPU infrastructure, automatic scaling. For startups and prototypes, the simplicity of OpenAI's embedding API is worth the cost premium.
  • Embedding lock-in is real: Switching embedding models requires re-embedding your entire corpus. Starting with a self-hosted model avoids dependency on any provider's pricing changes.
  • Domain-specific models outperform general ones: For medical, legal, or code search, fine-tuned open-source models outperform general-purpose OpenAI embeddings. The best model depends on your domain.
  • Dimensionality and cost trade-off: OpenAI's text-embedding-3 supports variable dimensions (256-3072). Lower dimensions save storage and compute in vector databases at minimal quality loss.
  • Batch processing at scale: Self-hosted models on a single GPU can embed millions of documents for a fraction of API costs. The ROI crossover happens quickly with large corpora.
