The Real Trade-offs

  • EventBridge has content-based filtering SNS doesn't: Filter on any JSON field in the event payload. SNS only filters on message attributes, not body content — a critical limitation.
  • SQS is still king for work queues: When you need guaranteed processing with visibility timeouts, dead-letter queues, and FIFO ordering — SQS is purpose-built. EventBridge is routing, not queuing.
  • EventBridge's 256KB payload limit hurts: SQS allows up to 256KB too, but with the Extended Client Library you can handle up to 2GB via S3. EventBridge has no such escape hatch.
  • Schema Registry is underrated: EventBridge auto-discovers event schemas and generates code bindings. This alone saves days of integration work across teams.
  • The right pattern is EventBridge + SQS together: Route with EventBridge, buffer with SQS. They're complementary, not competitors — but most teams pick one and suffer.
