S3 Events + Lambda: The Most Powerful Pattern on AWS

S3 events + Lambda is the most powerful pattern on AWS. File upload triggers processing, generates thumbnails, runs ML inference, indexes content — all without a single server. Yet most teams still poll S3 or use cron jobs.

The Real Trade-offs

  • S3 events are free: No additional cost beyond Lambda invocations. Compare this to polling S3 with ListObjects API calls every minute — that costs money and adds latency.
  • EventBridge integration unlocks advanced filtering: S3 to EventBridge lets you filter on object metadata, size, and prefix patterns with full content-based routing. Native S3 notifications only support prefix/suffix filters.
  • Duplicate events are the gotcha: S3 event notifications are at-least-once delivery. Your Lambda must be idempotent — processing the same file twice should produce the same result.
  • Fan-out needs SQS in the middle: S3 can notify one Lambda directly, but for multiple consumers, route through SQS or SNS. Direct fan-out from S3 to multiple Lambdas creates tight coupling.
  • Large file processing needs Step Functions: For files over 500MB, a single Lambda invocation may timeout. Use Step Functions to orchestrate chunked processing with parallel map states.
