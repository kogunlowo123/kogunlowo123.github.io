Lambda vs Fargate Cost: The Serverless Pricing Showdown

Everyone defaults to Lambda because "pay per invocation" sounds cheap. But when your function runs 50M times a month at 512MB for 3 seconds each, Fargate with a Savings Plan crushes Lambda on cost. The pricing crossover is lower than most teams realize.

The Real Cost Breakdown

  • Lambda's hidden multiplier — API Gateway: REST API Gateway costs $3.50/M requests. If your Lambda handles 10M requests/month, that's $35 in Gateway fees alone before you count a single millisecond of compute. HTTP API halves it, but it's still real money.
  • Fargate with ALB is a flat rate: An ALB plus a Fargate service running 2 tasks (0.5 vCPU, 1GB each) costs roughly $45/month. That handles millions of requests without per-invocation charges — and scales with target tracking.
  • GB-seconds compound fast: Lambda charges $0.0000166667 per GB-second. A 512MB function running 3 seconds per invocation at 10M calls = 15M GB-seconds = $250/month in compute alone. Fargate handles that for under $60.
  • Savings Plans tip the scale further: Fargate Compute Savings Plans offer up to 50% discount on committed spend. Lambda has no equivalent discount mechanism — you pay full price regardless of volume.
  • The real crossover point: For bursty, low-volume workloads under 1M invocations/month, Lambda wins hands down. Above 5M sustained invocations with >1s duration, model both — Fargate almost always wins.
