Serverless Framework vs SAM vs CDK: The Deployment Tool Debate

Serverless Framework pioneered the space and has the biggest plugin ecosystem. SAM is AWS-native with local testing built in. CDK lets you define infrastructure in TypeScript or Python. They all deploy CloudFormation under the hood — the difference is developer experience.

The Real Trade-offs

  • Serverless Framework: plugin ecosystem is the moat: Hundreds of plugins for offline testing, domain management, webpack bundling, and multi-provider support. But Serverless Inc. pivoted to their dashboard product — the open-source framework's future feels uncertain.
  • SAM: local testing is the killer feature: `sam local invoke` and `sam local start-api` let you test Lambda functions locally with Docker. Neither Serverless Framework nor CDK has anything this seamless for local development without third-party plugins.
  • CDK: real programming language, real power: Conditionals, loops, abstractions, type safety, and unit testing your infrastructure. Once you've used L2 constructs like `NodejsFunction` with automatic bundling, YAML feels primitive.
  • CloudFormation is the bottleneck for all three: They all generate CloudFormation templates. A 200-resource stack takes 15+ minutes to deploy regardless of which tool you use. CDK can't escape CFN's limits — 500 resources per stack, slow rollbacks, and cryptic error messages.
  • The pragmatic choice: New projects should start with CDK. Existing Serverless Framework projects should stay unless there's a compelling reason to migrate. SAM is ideal if local testing is your primary concern and your infrastructure is simple.
